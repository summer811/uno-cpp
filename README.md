# UNO-Cpp: A Custom-Art UNO Card Game Built with C++ and EasyX

## 📌 Project Overview
This project is the final assignment for the C++ Practical Course. We have implemented the core logic and visual interface of the classic tabletop game **UNO** from scratch using C++ and the EasyX Graphics Library.

⚠️ **Special Note**: To respect copyright and deliver a unique visual style, this project **does not use any official game textures**. All card assets and UI visual elements are **100% original and custom-drawn** by our team members. The project is now fully open-source for learning and exhibition purposes.

---

## ✨ Features & Highlights
* **Authentic Gameplay**: Complete implementation of core UNO rules, including standard number cards and all action cards (Skip, Reverse, Draw 2, Wild, and Wild Draw 4).
* **Original Custom Art**: Say goodbye to generic official assets. Features a unique, hand-drawn art style for a refreshing visual experience.
* **OOP Architecture**: Designed with clean Object-Oriented Programming (OOP) principles. Shuffling, dealing, play validation, and turn management are highly decoupled.
* **Smooth Interaction**: Powered by the EasyX library, offering intuitive mouse interactions and fluid animations instead of a dull console/CLI interface.

## 🚀 Quick Start & Build Guide

Follow the steps below to set up and run the project locally:

### 1. Prerequisites: Install EasyX
Since this project relies on EasyX for rendering, please ensure EasyX is configured for your Visual Studio 2022 before compiling:
1. Go to the EasyX Official Website (or [Click here to visit](https://easyx.cn/)) to download the latest installer.
2. Extract and run the installer, select **Visual Studio 2022**, and click install.

### 2. Clone the Repository
Open your Terminal or Git Bash and run the following commands:
```bash
git clone https://github.com/summer811/uno-cpp.git
cd uno-cpp
```
###3. Build and Run
Navigate to the project root directory and locate the [Project_Name].sln (Visual Studio Solution file).
Double-click uno-cpp.sln to open it in Visual Studio 2022.
In the top toolbar, ensure the build configuration is set to Debug or Release, and the platform is set to x64 (or x86, depending on your setup).
Press Ctrl + F5 (or click "Local Windows Debugger") to compile and launch the game!
